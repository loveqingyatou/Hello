工具简介

Github Pages

Hexo

软件安装

npm install -g hexo

博客搭建

Hexo发布博客的方式，是先在本地将博客编译成静态网页，再将静态网页提交至Github。所以本地需要有一个目录用于保存博客。比如，我们可以将博客的本地路径设置为~/Sites/blog/。将工作目录切换至~/Sites/blog/，然后执行如下命令来生成一个博客雏形。

# 初始化博客目录
hexo init
# 安装依赖
npm install

博客雏形生成之后，我们可以先睹为快！执行下面的命令，启动服务器，然后访问http://localhost:4000就可以看到默认生成的博客了。

# 启动本地服务器
hexo server

下面是几个Hexo的常用命令。

# 新建文章
hexo new post <title>
# 新建草稿
hexo new draft <title>
# 发布草稿
hexo publish <title>
# 生成静态页面
hexo generate
# 部署到Github
hexo deploy

用Hexo发布博客的的一个好处是可以使用自己喜欢的编辑器直接编写Markdown文件，写完后一个命令就发布到Github。同时因为Hexo生成的是静态页面，即使哪天Github因为你懂得原因无法访问，我们还可以轻易将博客部署到其他服务器上。当然最重要的本地博客目录要不能弄丢就是了。

博客定制

虽然默认生成的博客比较简陋，但是Hexo提供了很多可定制的地方，让用户可以获得足够的个性化空间。

页面风格

Hexo项目的Wiki页面有给出一个页面风格的集合。

用户执行如下命令，将页面风格克隆至本地目录，然后去修改_config.yml文件中的theme-name属性为想要的页面风格。

git clone <repository> themes/<theme-name>

例如我的博客使用的页面风格是hexo-theme-iOS7，其Git repo为https://github.com/tracy-e/hexo-theme-iOS7.git

我的当前目录为~/Sites/blog/，于是我先执行如下命令将该风格克隆至本地，

git clone https://github.com/tracy-e/hexo-theme-iOS7.git themes/hexo-theme-iOS7

然后修改~/Sites/blog/_config.yml中theme字段的值为hexo-theme-iOS7，即theme: hexo-theme-iOS7。

博客插件

Hexo项目的Wiki页面有给出一个插件的集合。

在本地博客目录下执行如下命令，可以安装、更新及卸载指定插件。

# 安装插件
npm install <plugin-name> --save
# 更新全部插件
npm update
# 卸载插件
npm uninstall <plugin-name>

Hexo有许多不同功能类型的插件。Generator类型的插件主要用于生成静态文件，比如hexo-generator-feed用于生成RSS订阅源，hexo-generator-sitemap用于生成站点地图。Renderer类型的插件主要用于渲染文本或者转换文本语法，比如hexo-renderer-dot用于渲染doT语法的页面模板，hexo-renderer-pandoc可以调用pandoc将用Markdown、Textile、reST等标记语言转换成html文件。Migrator类型的插件主要用于将博客从别的站点迁移至Hexo。Tag类型的插件主要用于快速在博文中插入内容，比如hexo-local-images可以让博文显示本地图片，省去了将图片上传至七牛等文件托管服务的麻烦，hexo-math提供Mathjax支持，可以在博文中方便地用 $LaTeX{}$ 语法书写数学公式，比如我们来写个欧拉公式玩玩。

$$e^{i\pi}+1=0$$
